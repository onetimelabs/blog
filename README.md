# README

The onetimelabs blog website, based on the [Reverie](https://jekyllthemes.io/theme/reverie) theme by Amit Merchant.

## Typesetting and Publishing

Please refer to the [contribution post](https://blog.onetimelabs.com/contributing/) in the blog.

## Post Planning and Tracking

Please use the following workflow to plan posts and track their status:

- To add a new planned blog post, please create a tracking issue in this repository: click on the `Add item` button at the bottom of the `Planned` column, and create a new issue in `onetimelabs/blog` (you can press `#` to quickly select the repository). If your post is already in writing, please use the `Being Written` column instead; if your post is already written and ready for review, please use the `Ready` column instead.
- To describe what the blog post is about, please use the description of its tracking issue.
- To track progress on a blog post, please use comments on its tracking issue. In particular, please include a link to the markdown file (in the relevant repository branch) when the post is ready for review, and include a link to the post on [blog.onetimelabs.com](https://blog.onetimelabs.com) once the post is published. 
- To indicate that a post has been published, please mark the issue as closed (it will be moved to the `Published` column automatically).
